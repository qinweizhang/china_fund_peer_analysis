> 版本：1.2　|　更新日期：2026-07-09
> 适用项目：同业基金分析网页平台（china_fund_peer_analysis）
> 第一阶段 demo 已落地，迭代更新见 §10.1。

---

## 1. 项目背景与目标

本项目源自风险管理部《权益竞争对手季度持仓及规模变动分析》报告，原以 PPT 形式呈现。目标是将其改造为一个**可交互的网页分析平台**，把基础数据经相应业务逻辑加工后，以网页图表的形式呈现同业基金公司的业绩、持仓与规模变动情况，供我司（工银瑞信）与同业对标分析。

报告样本口径：选取 **2026 年一季度偏股基金规模前十五大基金公司** 作为分析对象，关键对标公司为**永赢**（规模爆发型）与**大成**（头部基金经理压舱石型）。我司权益规模排名第十一（1068 亿元）。

平台分三大模块：

1. **同业业绩及规模概况**
2. **同业持仓及变动分析**
3. **同业规模及变动分析**

### 2.2 数据口径约定（来自报告方法论）

- **持仓对比**：本季度与上季度均**只考虑各基金前十大重仓股**；采用对应所有产品股票市值加总计算占比。
- **规模变动拆分**：按当季业绩涨幅、新发拆分后，**剩余变动归为持营**；当季新发基金规模 = 季末时点单位净值 × 发行份额估算。
- **板块**：TMT、金融地产、消费、新能源及制造、周期、医药六大板块 + A股/港股市场拆分。
- **事后分类填补逻辑**：产品规模表中 CLASSIFY_LABEL 需用"产品事后分类"INDUSTRIESNAME 匹配；匹配为空时按既定规则填补（见 Excel 注释，开发时落地该逻辑）。

## 3. 三大模块需求详述

### 模块一：同业业绩及规模概况

呈现十五家基金公司权益规模与长短期业绩的总览。

- **主表：同业十五家基金公司权益规模及业绩情况**
  - 列：序号、基金公司、2025Q4 末规模、2026Q1 末规模、规模变动幅度、2026Q1 规模变动拆分（收益率上涨 / 持营 / 新发，单位亿元）、2026 年投资收益率（含十五大排名、同业排名）、2024–2026 年投资收益率（含十五大排名、同业排名）。
  - 排序：2026Q1 业绩降序。我司（工银瑞信）行需高亮标识。
  - 数据来源：权益银河排名 + 产品规模。

## 4. 技术选型

| 层 | 选型 | 说明 |
|----|------|------|
| 前端框架 | **Flask（Python）** | 替代 ADLS 文档中的 NPM/Node 方案。用户可读、可维护，由大模型生成代码。不引入复杂前端构建链。 |
| 模板 | Jinja2 | Flask 内置，渲染三大模块页面。 |
| 数据处理 | pandas + openpyxl | 读取/加工 Excel，计算规模变动拆分、收益率、最大回撤、波动率等。 |
| 图表 | ECharts 或 Plotly（JS 端） | 折线、散点、堆叠面积、热力表格；与 ADLS 主题色一致。 |
| 数据表格 | AG Grid 风格 / 原生 table | 高密度数据表，等宽数字，微边框。 |
| 样式 | Tailwind CSS（CDN 或本地） | 按 ADLS 配置注入深海色板与直角规则。 |

**理由**：完全照搬 ADLS 原版会产出 NPM 代码，超出维护者能力范围；本项目仅借鉴 ADLS 的**视觉与交互设计元素**（主题色、界面分布、模块样式、组件规范），底层用 Flask 实现。

## 5. 设计规范（遵循 ADLS v2.1）

详见 `reference/ADSL.md`，要点：

- **绝对正交**：所有元素 `border-radius: 0`。
- **微边框主义**：用 1px 边框与色块替代投影；主边框 `border-slate-300`，次级 `border-slate-200`。
- **完全去背**：全局底色与内容面板均 `Slate-50 #F8FAFC`，**仅输入框为纯白**（白色=可编辑心智模型），边框加深补偿。
- **极高密度**：默认 12–13px 字号，`leading-tight`，等宽数字 `font-feature-settings: "tnum"`。
- **深海色板**：主色 `#205781`（深海蓝）、辅色 `#4F959D`（冷杉绿）、文字 `#334155/#64748b`；涨绿 `#10b981`、跌红 `#ef4444`（A 股红跌绿涨，可配置）。
- **容器（Bento Box）**：Slate-50 底 + 1px slate-300 边框 + 32px slate-100 标题栏，无圆角无阴影。
- **数据表格**：Header `bg-slate-50`/`text-xs font-bold text-slate-500`，Cell `text-xs font-mono tabular-nums`，选中行 `bg-able-mint/10`。
- **图表透明化**：图表直接渲染在 Slate-50 上，不设白色背景。

## 6. 页面与交互架构

- **导航**：顶部主导航切换三大模块（按键放大，字号 15px / 高度 52px）；模块内左侧/顶部二级导航切换子表/子图。
- **数据集切换与上传**：顶栏右侧数据集下拉切换 `input_data/` + `uploads/` 下任意 Excel；"＋ 上传数据"按钮上传新季度 Excel（与示范同结构 12 sheet），自动登记并切换为激活。季度与最新时点由数据派生，跨季度通用。
- **公司筛选**：支持公司多选/全选，默认展示十五大 + 我司；图表随筛选联动（第二阶段）。
- **时间维度**：持仓与规模历史图支持区间选择（第二阶段）。
- **下钻**：表中公司名/基金名可下钻到明细页（第二阶段）。
- **导出**：表格支持导出 Excel/CSV（第二阶段）。
- **响应式**：以桌面端宽屏为主（分析师终端场景），不做移动端优先适配。

## 7. 开发阶段规划

### 第一阶段：静态数据展示（MVP）
1. 搭建 Flask 项目骨架（路由、模板、静态资源）。
2. 落地 Excel 读取层：按 sheet 封装加载与基础清洗，建立数据字典。
3. 实现三大模块页面的静态渲染：模块一主表、模块二四张配置表 + 两张重仓股表、模块三规模拆分表 + TOP10 表，按 ADLS 样式呈现。
4. 渲染历史折线图（集中度、仓位）与规模/业绩散点图、堆叠面积图。
5. 我司行高亮、红绿条件着色等基础可视化规则。

### 第二阶段：交互与动态优化
1. 公司筛选、时间区间选择、子模块联动。
2. 表格排序、筛选、下钻到明细页。
3. 规模变动拆分、最大回撤/波动率等指标的计算逻辑上线（基于产品复权净值）。
4. 导出功能、加载性能优化（数据缓存/预聚合）。
5. 细节口径校对（事后分类填补、剔除行业基金口径、前十大重仓股口径）与报告数值一致性验证。
